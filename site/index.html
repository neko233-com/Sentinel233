迁移路径

先并行，后接管。保持 exporter 和 agent 不动，把控制面、面板和规则逐步迁入 Sentinel233。

01

接入数据

将 Prometheus Agent、Grafana Agent 或 Alloy 的 remote_write 指向 Sentinel233。

02

导入配置

通过 compat importer 导入 Prometheus scrape config、rules 和 Grafana datasource。

03

迁移面板

导入 Grafana dashboard JSON，保留变量、多 target、layout 与 fieldConfig。

04

下线 Grafana

通过 Docker E2E 和 CI 验证后，把查询入口与告警入口切到 Sentinel233。

迁移命令

以下命令就是 Docker E2E 中覆盖的同一组入口，可直接用于灰度环境 rehearsal。

sentinel233 replacement rehearsal
docker compose up -d --build

curl -X POST http://127.0.0.1:23390/api/v1/write \
  -H "Content-Encoding: snappy" \
  -H "Content-Type: application/x-protobuf" \
  --data-binary @prometheus-write-request.snappy

curl -X POST "http://127.0.0.1:23390/api/compat/import?source=prometheus-config" \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/yaml" \
  --data-binary @prometheus.yml

curl -X POST "http://127.0.0.1:23390/api/compat/import?source=prometheus-rules" \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/yaml" \
  --data-binary @rules.yml

curl -X POST http://127.0.0.1:23390/api/dashboards/import \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  --data-binary @grafana-dashboard.json

pwsh ./scripts/docker-grafana-replacement-e2e.ps1

Docker 全链路验证

脚本会构建镜像、启动 Compose、登录、导入生态格式、写入 remote_write 样本、查询 PromQL、验证 Grafana 变量端点、导入并导出 Grafana dashboard、接收 Alertmanager webhook。

1Docker BuildKit cache 加速模块下载与 Go build。
2Compose 使用 project name 隔离，支持 CI 与本地反复执行。
3脚本失败即停，输出具体失败步骤。
4CI 可直接调用同一脚本，避免“本地可以、流水线不行”。
